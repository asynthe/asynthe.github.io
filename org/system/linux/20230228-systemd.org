* services

[[https://superuser.com/questions/1423672/systemd-service-not-executing-the-script][systemd service not executing the script? - StackExchange]]

Create a service in ~/etc/systemd/system/<name>.service~

Start the service
~$ systemctl start <service>~

Check the status
~$ systemctl status <service>~

* units/services checking

[[https://www.cyberciti.biz/faq/systemd-systemctl-list-all-failed-units-services-on-linux/][Systemd systemctl list all failed units/services on Linux - nixCraft]]

~# systemctl list-units --failed~
or
~# systemctl list-units --state failed~

* timers
:PROPERTIES:
:ID:       931ab427-e80f-453e-8274-ac185e02f626
:ROAM_ALIASES: "systemd timers"
:END:

[[https://nixos.wiki/wiki/Systemd/Timers][Systemd/Timers - nixos wiki]]
[[https://www.freedesktop.org/software/systemd/man/systemd.timer.html][systemd.timer man page - freedesktop.org]]

Check for your service _timer_ with:
~$ systemctl list-timers~

You can use [[id:532f1c61-926c-475d-9690-573120aedf3e][watch]] to monitor it:
~$ watch -n 1 systemctl list-timers~

+ Remember that the *service unit* _must be the same_ as the *timer unit*.

* journalctl

[[https://linuxhandbook.com/journalctl-command/][Beginner's Guide to Analyzing Logs in Linux With journalctl Command - LINUX HANDBOOK]]
[[https://www.shellhacks.com/journalctl-tail-service-logs-systemd-journal/][Journalctl: Tail Service Logs - Systemd Journal - ShellHacks]]

Monitor events on your system.
~# journalctl -f~
_note_: Using ~tail~ will not work on, we can use one of it's flags.

Monitor a specific service
~$ doas journalctl -u <service> -f~
_example_:
~$ doas journalctl -f -u swww-wallpaper.service~
