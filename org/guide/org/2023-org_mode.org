** timestamps

a way to add timestamps to different text or TODOs is using: ~org-time-stamp~ (*Emacs*: , *Doom*: SPC m d t)

this will prompt you a calendar in which you can move with ~SHIFT + ARROW KEYS~ or the ~Mouse~

- you can set a time such as ~11:00~ or ~11am~ in the prompt, it will format it for you.

using ~SHIFT~ on the ~<>~ will change them from:
- <2023-03-04 Sat> ACTIVE
- [2023-03-04 Sat] INACTIVE

using ~SHIFT + ARROW KEYS~ while cursor is on the /date/ will change the /date/

** checkboxes

using checkboxes is easy. you do them with the brackets in a list and they commonly have three states:
- [ ] empty
- [-] in progress
- [X] done

use ~C-c C-c~ (emacs way) or ~SPC m x~ (doom way) to tick them.

checkboxes can have a /number/ or /percentages/ if it has c
a listed item with a checkbox can have children, and have a completed /number/ or a completed /percentage/

- [-] go to the gym
  - [X] do 25 pull-ups
  - [ ] do 50 push-ups

- [50%] go to the gym
  - [X] do 25 pull-ups
  - [ ] do 50 push-ups

a parent can be added the number of done checkboxes or the percentage.

***** TODO go to supermarket [1/3]
- [X] buy milk
- [ ] buy a banana
- [ ] pet a cat

***** TODO go to the supermarket [33%]
- [X] buy milk
- [ ] buy a banana
- [ ] pet a cat

https://wmealing.github.io/emacs-org-babel-analysis.html

[[https://leanpub.com/lit-config/read][Literate configuration - Leanpub]]
[[https://en.wikipedia.org/wiki/Literate_programming][Literate Programming - wikipedia page]]

1. Create ~README.org~, so you can use it in Github.
   Inside of it, write the org metadata.
   _example_:
   ~#+title: Xmonad configuration
   ~#+property:~

2. Copy/paste and turn your config into a literate config
3. When done, change

   ~#+property: header-args :tangle xmonad.hs~

   ~#+property: header-args~ -> Apply to every single source code block, globally.
   ~:tangle~ -> write it to *this* file 

4. Refresh the file
   Put your cursor on ~#+property:~ and do a ~C-c C-c~ so it does a local refresh. (should appear on your modeline)

5. Time to create the file
   Do a ~M-x org-babel-tangle~ or ~C-c C-v t~

6. Now just push your ~README.org~ to Github or Gitlab!

_note_: check also for org-auto-tangle, which can help when writing literate configs.

*** usage

open org-agenda
~M-x org-agenda~
*Emacs*: ...
*Doom*: ~SPC o A~

use org-schedule
~M-x org-schedule~
*Emacs*: ~C-c C-s~
*Doom*: ~SPC m d s~

mark heading as todo
*Doom*: ~SPC m t t~

if you want to see more days than the normal 10 days shown at Org Agenda, you can press a number + org agenda keybind:
_example_: 21 days
~21 SPC o A~

*** Setting it up

is it a tool that helps you to group in a single page all TODOs that are attached to notes with certain tags.

open your ~org-agenda~ with SPC o A

set up your org agenda main file with: (in ~~/.doom.d/config.el~)
#+begin_src elisp
(after! org
  (setq org-agenda-files
        '("~/main/sync/docs/org/agenda.org")))
#+end_src

now, add TODOs and headings on your agenda file, but keep in mind that this one will not appear on ~org-agenda~ until they have [[id:05ed4a55-ee90-4500-b146-ae1d9fe79c3e][Timestamps]]

you can also add [[id:39a19b80-c502-42fa-a281-2f19f583958b][Priorities]] on your TODOs, and they will appear on ~org-agenda~

*** Making a custom view-mode
*** Changing some characters

now, in our new view-mode of org-agenda (SPC o A v)
we will see that the separator character for the different tasks priorities will be ~=~, which does not look pretty

we can change this with
#+begin_src elisp
(setq org-agenda-block-separator 61)
#+end_src

you can set up a character but it's better to put it's code.
We can find it using ~describe-char~ (SPC h ') while the cursor is on the symbol, ~codepoint~ will tell what codes we can use.

in this case i will use the simple dash, which code is ~45~.
#+begin_src elisp
(setq org-agenda-block-separator = 45)
#+end_src

now, restart your doom emacs config (SPC h r r)
