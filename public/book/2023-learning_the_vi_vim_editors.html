Opening and Closing Files

The editor copies the file to be edited into a buffer1, displays the buffer (but you can see only one screenful at a time), and lets you add, delete, and change text.

When saving edits, the editor copies the edited buffer into a permanent file, replacing the old file.

Remember that you are always working a copy of your file in the buffer, your edits don't affect original file until you save the buffer, that's why saving the edit is called "writing the buffer" or "writing your file."

  1. An area temporarily set aside in memory

Opening a File from the Command Line

The vim command: $ vim [filename] or $ vi [filename]

the file is optional.

  • if the file is in the same directory, you can directly open vim with the filename.
  • if it's on another directory, you will have to specify it from the relative pathname, or with the dots. $ vi /home/john/letter

Problems Opening Files

  • terminal: Unknown terminal type

set $TERM to vt100, can temporarily help.

  • A [new file] appears when a file already exists. Check for correct case in the filename.
  • You get into ex line-editing mode. (you get a colon prompt) You probably typed an interrupt (typically CTRL-C) before vi draws the screen, type vi at the ex prompt (:).
  • One of following messages appear:
    • [Read only] Means you cannot save any changes you make.
      • May have invoked vi in view mode (with view or vi -R)
      • Don't have the write permission for the file.
  • Following messages
    • Bad file number
    • Block special file
    • Character special file
    • Directory
    • Executable
    • Non-ascii file
    • file non-ASCII

The file is not a regular text file

Modus Operandi

two modes

  • command mode
  • insert mode

ex command can be a third mode.

The editor may beep if you exit two times. Use this to confirm you've exited wherever mode you're in. You exit with ESC.

Saving and Quitting a File

The command to quit and save edits is ZZ.

the ex commands are also used, these are:

  • :w to save (write) but not quit
  • :q to quit (if you haven't made any edits)
  • :wq to both save edit and quit. (:wq is equivalent to ZZ.)

Quitting Without Saving Edits

  • :e! will Wipe all edits and reload original file
  • :q! will Wipe all edits and quit vim.

Problems Saving Files

You get any of the next messages: File exists File <filename> exists - use w! [Existing file] File is read only

Type :w! to overwrite the existing file, or type :w <new-filename> to save into a new file
