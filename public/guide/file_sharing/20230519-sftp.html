MOVING BETWEEN THE SYSTEMS

We are going to be managing two filesystems while using sftp.

  • The Local directory (your computer).
  • The Remote directory.

So the way sftp does to manage both systems is to have two versions for the commands, example:

  • cd: will move around the remote machine directories.
  • lcd: will move around the local machine directories.

Give it a try! Check you are moving between the two filesystems with:

  • ls: for the remote machine.
  • lls: for the local machine.

COPYING A FILE

PUT - LOCAL TO REMOTE

  • Change to local directory with the files
  • Change to remote directory where you want to send

Copy a file like this: sftp> put <filename> sftp> put <file1> <file2> Copy a directory: sftp> put -r <directory> Copy all, files and directories: sftp> put -r *

Now, if a file transfer fails or is interrupted, resume it with reput sftp> reput <filename>

GET - REMOTE TO LOCAL

  • Change to local directory with the files
  • Change to remote directory where you want to send

Get a file from the remote machine like this: sftp> get <filename> sftp> get <file1> <file2> ...

Get a directory: sftp> put -r <directory>

Get all, files and directories: sftp> get -r *

Now, if a file transfer fails or is interrupted, resume it with reget sftp> reget <filename>
