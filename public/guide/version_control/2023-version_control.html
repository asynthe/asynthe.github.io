DOTFILES

THE BASICS OF DOTFILES

from The Basics of Dotfiles - System Crafters (youtube video)

What are they?

  • Text-based configuration files for the programs you use.
  • Personal scripts.
  • Called "dotfiles" because the file or directory starts with "."

Common dotfiles

  • ~/.bash_profile - A script loaded when you log in.
  • ~/.bashrcc - A script loaded when you open a terminal.
  • ~/.emacs.d - Emacs configuration.
  • ~/.vimrc - Vim configuration.
  • ~/.gitconfig - Configuration for Git.
  • ~/.config - A folder full of many more configuration files.

In Windows case, many cross-platform programs will store in C:\Users\username or C:\Users\username\AppData\Roaming.

How should I manage them? The best way to manage your dotfiles is to keep them in source control.

Benefits

  • Easily sync your configuration between multiple machines
  • Roll back to a previous config
  • Experiment with huge changes w branches
  • Have a log of the changes

Which program? It doesn't matter, decentralized is better!

  • Git
  • Mercurial (implications for which host you can use)
  • Darcs

Publish your dotfiles!

It feels scary, but it's worth it.

What's next?

  • Manage and share your dotfiles with Git
  • Tools to make dotfiles management easier like GNU Stow
  • Complete dotfiles management with Emacs (even on Windows!)

HOW TO CREATE A DOTFILES FOLDER

from How to Create a Dotfiles Folder - System Crafters (youtube video)

  • Create a folder in the $HOME directory, or as close as you can.
  • Move some configuration files into it.
  • Link them back to their original location
  • Verify that it works!

System Crafters recommends this directory, as is it easier to use tools like GNU Stow: $ mkdir ~/.dotfiles

Create symbolic links to the original config file locations

$ ln -sf ~/.dotfiles/.emacs.d ~/.emacs.d

DOTFILES SYMLINKS IN WINDOWS

In windows create a junction using mklink. important: You need a elevated prompt. (Run cmd as administrator)

Create a link for a individual file with the /H flag. $ mklink /H <link-name>.conf <original-file>.conf

Create a link for a directory with the /J flag. $ mklink /J <link-name> <original-folder>

example: Emacs configuration. $mklink /J c:\Users\benjamin\AppData\Roaming\.emacs.d c:\Users\benjamin\AppData\Roaming\.dotfiles\.emacs.d

THE PROBLEM BETWEEN MACHINES

It's tedious to create and manage symlinks like this when having different computers. Some people solve this by creating a "bootstrap" script that creates all symbolic links automatically.

It's easier to use a tool made for this, such as GNU Stow and others.

GNU STOW

It describe itself as a "symlink farm manager".

It's a program that can mirror the structure of one directory into another by creating symbolic links back to the original files.

Extremely useful when you have a directory with configuration files (~/.dotfiles or other) managed by git, and you want to send all of this configuration files (symlink) into where they belong in your home directory.

install

arch -> gentoo -> nixos ->

Windows -> Install Stow via the MSYS2 command shell using pacman -S stow.

  • Make sure you run the shell as Administrator.
  • Have the MSYS variable set to winsymlinks:nativestrict, so that it creates _symbolic links correctly, if not, stow will just copy the files.
