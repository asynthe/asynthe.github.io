usage

—

system for managing your files

start a github repo $ git init

in .gitignore add the files you want git to ignore and not upload to the repository

stage the files, to include in a commit $ git add . or $ git add <filename>

unstage files with $ git reset . or $ git reset <filename> --hard -> will unstage and delete the files

commit files $ git commit -m "first commit" you can make your comment be multiline $ git commit -m "first commit > This is my first commit > "

branches

create a branch $ git checkout -b feature

stash

save current changes without commiting them, then get back to a clean working directory $ git stash -u

get back to work with $ git stash pop

updating a git repo

So, you have this git repo and to update is it as simple as doing $ git pull

Now, to update git repos that are on subdirectories do this: $ git fetch --recurse-submodules

So finally: $ git fetch --recurse-submodules && git pull
