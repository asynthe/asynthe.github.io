RESOURCES

I think "compatibility layer" is the more technical term for what Wine actually is, as it converts low-level Windows system calls to something that the Linux (and other UNIX/POSIX) kernels can understand and execute, and effectively runs the application "natively" (or semi-natively?). It's equivalent to the old code in 32-bit versions of Windows that allowed MS-DOS programs to run, or Windows' own settings to allow programs to run in "compatibility mode" for older versions of the OS.

Wine does not emulate an entire CPU like, for example, DOSBox, BSNES and KEGA Fusion… so therefore, "Wine is not an emulator." It just translates things meant for the Windows kernel in such a way that other kernels can understand.

note: always run as user. note: you can use Bottles which is a easier way to manage Wine apps.

WINECFG - WINE CONFIGURATION Now configure your Wine subsystem $ winecfg (will open the configuration for ~/.wine.) $ WINE_PREFIX=~/wine/<folder> (will open the configuration for ~/wine/<folder>.)

INSTALLING AN .EXE To run a Windows .exe on Linux, open it like this. $ wine <installer>.exe or $ WINE_PREFIX=~/wine/<folder> wine <installer>.exe

UNINSTALLING To uninstall any Windows software, open terminal and bring a Windows-style control panel. $ wine uninstaller or $ WINE_PREFIX=~/wine/<folder> wine uninstaller

RUNNING You will have to go into the Wine folder and find the application executable. It will probably be in the "Program Files" or "Program Files (x86)" in the prefix created C drive. ~/.wine/drive_c

Go into that directory and simply run it like this: $ wine program.exe or $ WINE_PREFIX=~/wine/<folder> wine program.exe

note: WINE IN DIFFERENT NIXOS ARCHs

on x86_64-linux wine supports by default 32-bit and 64-bit applications. In other architectures, it only supports 32-bit applications.

If you're using another architecture like RISC-V, you can override the Wine package so it includes 64-bit applications. 

{
  environment.systemPackages = with pkgs; [

    # support both 32- and 64-bit applications
    wineWowPackages.stable

    # support 32-bit only
    wine

    # support 64-bit only
    (wine.override { wineBuild = "wine64"; })

    # wine-staging (version with experimental features)
    wineWowPackages.staging

    # winetricks (all versions)
    winetricks

    # native wayland support (unstable)
    wineWowPackages.waylandFull
  ];
}

In other architectures, replace this line 

# support 64-bit only
(wine.override { wineBuild = "wine64"; })
