NETWORKING

By default, VMs will only have network access to other VMs on the same server (and to the host itself). If you want the VMs to have access to your VLAN, then you must create a network bridge on the host as explained here.

Edit /etc/sysconfig/network-scripts/ifcfg-eth0 and add the line BRIDGE=br0 (make sure to remove any static IPs).

Create the file /etc/sysconfig/network-scripts/ifcfg-br0 and add the entries as shown below. You can use static or dhcp. I used static IP in this case. 

DEVICE="br0"
BOOTPROTO="static"
IPADDR="xxx.xxx.xxx.xxx"
NETMASK="255.255.255.0"
ONBOOT="yes"
TYPE="Bridge"
NM_CONTROLLED="no"

If you are running a firewall, (like iptables), create a rule in to allow bridged traffic.

ENABLE IP FORWARDING Enable IP forwarding in /etc/sysctl.conf by adding the following line: 

inet.ipv3.ip_forward=1

During the testing phase, if you have SELinux running, disable it by setting SELinux to permissive mode.

Reboot the system after the above changes are done.

CREATING A VM (VIRT-INSTALL)

FLAGS

VM OPTIONS --name / -n -> name of virtual machine --description -> Some description about your VM. example: Application server, database server, web server, etc.

--os-type -> DEPRECATED, DO NOT USE --os-variant -> Distribution type for the above os-type.

  • For linux, it can be rhel6, centos6, ubuntu14, suse11, fedora6 , etc.
  • For windows, this can be win2k, win2k8, win8, win7
  • Check for a distribution with $ virt-install --osinfo list

-c / --cdrom -> The location of the installation image. -l / --location -> Distribution tree installation source.

note: it's relative to your current directory unless an absolute path is specified.

--disk -> location of created VM file path: where the VM image files is stored. size: in GB format: qcow2

HOST OPTIONS --cpu -> x --virt-type -> x

GUEST OPTIONS --boot -> uefi --vcpu / --maxvcpus -> number of virtual CPU's for guest --memory / --maxmemory -> memory in MB --ram: Memory for the VM in MB

--graphics / --nographics -> none, vnc

  • none: This instructs to use a text console on VM serial port instead of graphical VNC window. If you have the xmanager set up, then you can ignore this parameter.

--network bridge:br0: This example uses bridged adapter br0. It is also possible to create your own network on any specific port instead of bridged adapter.

If you want to use the NAT then use something like below for the network parameter with the virtual network name known as VMnetwork1. All the network configuration files are located under /etc/libvirt/qemu/networks/ for the virtual machines. For example: --network network=VMnetwork1

VIRTUALIZATION OPTIONS -v, --hvm -> Fully virtualized guest -p, --paravirt -> Paravirtualized guest --container -> Containerized guest --virt-type -> Hypervisor to use (kvm, qemu, xen, …) --arch <arch> -> CPU architecture to simulate --machine <machine> -> Machine type to emulate

EXAMPLE - RHEL

as root 

virt-install \
-n myRHELVM1 \
--description "Test VM with RHEL 6" \
--os-type=Linux \
--os-variant=rhel6 \
--ram=2048 \
--vcpus=2 \
--disk path=/var/lib/libvirt/images/myRHELVM1.img,bus=virtio,size=10 \
--graphics none \
--cdrom /var/rhel-server-6.5-x86_64-dvd.iso \
--network bridge:br0

EXAMPLE - OPENBSD

as root: 

virt-install \
--name=openbsd \
--virt-type=kvm \
--memory=2048,maxmemory=4096 \
--vcpus=2,maxvcpus=4 \
--cpu host \
--os-variant=openbsd5.8 \
--cdrom=/var/lib/libvirt/boot/install63.iso \
--network=bridge=virbr0,model=virtio \
--graphics=vnc \
--disk path=/var/lib/libvirt/images/openbsd.qcow2,size=40,bus=virtio,format=qcow2

open # vncviewer

get port # virsh dumpxml openbsd-srv | grep vnc

MANAGING THE VM WITH VIRSH

How to delete KVM VM guest using virsh command - nixCraft

Connect to the vm # virsh console <vm-name> Exit out of the console with Ctrl + ] note:

Start # virsh start <vm-name>

Reboot # virsh reboot <vm-name>

Shutdown # virsh shutdown <vm-name>

Force stop # virsh destroy <vm-name>

Delete the vm # virsh undefine <vm-name> note: vm file is not deleted by command, nor save images and snapshots. --remove-all-storage -> delete the vm file --managed-save -> delete managed save images --snapshots-metadata -> remove snapshots of vm fix: cannot undefine domain with nvram run # virsh undefine --nvram <vm-name>

Display vm cpu and memory usage # virt-top

Diplay vm information # virsh dominfo <vm-name>

List the virtual machines that are installed on your OS. # virsh list -> the ones that are running. # virsh list --all -> all the installed vms.

To edit a VM settings, the configuration file will be on /etc/libvirt/qemu/<vm>.xml However, avoid editing the file manually, use the next command: # virsh edit <vm-name>

Get the guest's IP address You can do inside the guest with $ ip a, but it can also be done from the host by using the next command: # virsh net-dhcp-leases <network> network -> default, br0

In case the network is not running, nor you are detecting any devices, start it like this: # virsh net-start default

Start vm when host starts # virsh autostart <vm-name>
