Network File System

  • NFS is not encrypted:

Use [Kerberos] or [Mullvad] (VPN) if sharing sensitive data.

  • NFS does not have user authentication by default:

client access is restricted by IP/hostname.

  • User and/or user group IDs should be the same on both client and server
    • Enable NFSv4 idmapping
    • Overrule UID/GID by using anonuid/anongid + all_squash in /etc/exports.
  • POSIX ACLs are not supported.

BusyBox mount command doesn't support NFSv4
