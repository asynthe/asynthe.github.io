ATA Secure Erase with HDPARM

https://archive.kernel.org/oldwiki/ata.wiki.kernel.org/index.php/ATA_Secure_Erase.html

Avoid's wear and tear, but will completely delete everything on the SSD

"An ATA secure-erase command can only erase the entire disk (not just a partition, not just a file). On SSDs with full-time encryption, it works by generating a new key and overwriting the old one, so nothing in the flash cells can be decrypted correctly anymore. (Also clearing the block mapping data so the whole disk acts as freed). This is ideal if you're going to sell a drive, if it supports instant-erase you avoid any wear and tear on the flash."

helpful packages

arch -> . gentoo -> sys-apps/nvme-cli and sys-apps/hdparm nixos -> nvme-cli and hdparm

Kernel I/O scheduler

see the current I/O scheduler on your system like this $ cat /sys/block/nvme0n1/queue/scheduler

it may have multiple I/O schedulers available, such as [none] mq-deadline bfq

you can change the scheduler with an echo # echo "none" > /sys/block/nvme0n1/queue/scheduler

if it doesn't work, make a udev rule. in /etc/udev/rules.d/60-ioschedulers.rules 

# Set sheduler for NVMe devices
ACTION=="add|change", KERNEL=="nvme[0-9]n[0-9]", ATTR{queue/scheduler}="none
