configuration

show the current configuration

HOSTNAME#show run HOSTNAME#show ip int br

saving current configuration

SAVING HOSTNAME(config)#ex ~HOSTNAME#copy running-config startup-config HOSTNAME#ex

note: shorter version HOSTNAME# copy run start

TFTP - backup / restore

BACKUP Router#copy running-config tftp: Adress or name of remote host []? <number> Destination filename [router-confg]? <filename>

RESTORING Router#copy tftp: running-config Adress or name of remote host []? <number> Destination filename [router-confg]? <filename>

erase startup configuration

HOSTNAME>en HOSTNAME#write erase HOSTNAME#erase startup-config HOSTNAME#reload

change hostname

HOSTNAME>en HOSTNAME#config t HOSTNAME(config)#hostname <hostname>

note: remember to save!

updating the firmware

Understand how to Backup and Restore Configuration Files - Cisco Docs

You first have to go into https://cisco.com > Support > Download and grab the ISO image you need

Verify current version: Router>show version

Check if you have enough space left on the Router memory: Router>show flash

Then we will have to choose the options of how to copy the ISO image to our memory

  • TFTP
  • FTP
  • SCP
  • USB stick (shows as usbflash)

In this case we will use TFTP Router#copy tftp flash

Address or name of remote host []? <TFTP Server IP Address> Source filename []? <Firmware filename> Destination filename [router]? <Destination filename>

Confirm the firmware update: Router#show flash

Set the new firmware as the default boot image: Router(config)#boot system flash <Destination filename> Router(config)#exit Router#write memory

Reload the router to apply the firmware update: Router#reload Proceed with reload? [confirm]

passwords / password encryption

note: If password is not encrypted, it can be seen with show running-config / show startup-config. note: A Secret encrypts the Main / Privileged Password, in the case of the others (con, aux, vty) we need to activate the service password.

Console: Used for console access. Auxiliary: Used for the auxiliary port (if router/switch has one.) VTY: Used for Telnet and SSH access to the router/switch.

ENABLE PASSWORD Router>en Router#config t Router(config)# enable password <password> # Password Router(config)# enable secret <secret> # Secret password

CONSOLE LINE PASSWORD Router(config)#line 0 Router(config-line)#password <password> Router(config-line)#login Router(config-line)#ex

AUXILIARY PASSWORD Router(config)#line aux 0 Router(config-line)#password <password> Router(config-line)#ex

TELNET (VTY) PASSWORD Router(config)#line vty 0 4 Router(config-line)#password Router(config-line)#password ICT@Stanley! Router(config-line)#login Router(config-line)#ex ~Router(config)#

PASSWORD ENCRYPTION Router(config)#service password-encryption

banner

Custom message displayed when connecting to a Cisco IOS device Note: don't have a delimeter ($) on the message as cisco ios will interpret it as indicator to end banner message.

MOTD -> Login -> (user logs in) -> Exec

MOTD Router(config)# banner motd $<message>$

LOGIN Router(config)# banner login ?<message>?

EXEC Router (config)# banner exec #<message>#

note: the symbol chosen as starting the message will be the one to also close the message, so be careful not to include it in it. note: omitting the symbol, you can add more lines. example: Authorized personnel only.
