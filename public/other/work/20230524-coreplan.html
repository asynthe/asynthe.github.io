The CorePlan CSV Template

The CSV has various rows (horizontal) and columns (vertical)

FIRST THREE COLUMNS - field_id, name, page field_id = unique identifier for the field itself (keep IDs short and acronym-based, for easy reference) name = title of the field that the user sees page = title of the page the field appears on

FOURTH COLUMN - type type = define type of input, can be any of the following.

  • number = simple number value, example: hours.
  • text = small text for short responses, example: name.
  • textarea = large text for extended responses, example: comments.
  • checkbox = Yes or No response field.
  • select = Multiple choice field with selectable options.
  • geolocation = GPS coordinate. (default: current device location)
  • date = A calendar date without time. (DDMMYYYY?)
  • time = A time without date. (HH:MM:SS?)
  • datetime = Both date and time. (HH:MM:SS,DDMMYYYY?)
  • rig-select = Active rigs as drop-down options.
  • people-select = Active users and personnel as drop-down options.
  • info = Text-only info message with no input
    • required must be FALSE.
    • help_text is MANDATORY.
  • signature = Signature input
  • lookup-select = Insert a lookup list
    • Funtions as a select type (multiple choice field)
    • List name must be specified within lookup-list column
    • Select choices are retrieved from a choosen lookup-list within the lookup-list library
    • Able to modify the list directly in CorePlan.

FIFTH COLUMN - required

TRUE or FALSE, depending on MANDATORY or not field (is this REQUIRED?)

SIXTH COLUMN - is_multi

If you want user

  • to select ONE CHOICE: FALSE
  • to select more than ONE CHOICE: TRUE

note: only use if type is either: select, people-select, rig-select or lookup-select

SEVENTH COLUMN - help_text

A small subtext under the field name.

EIGHTH COLUMN - help_image

Insert a form help image that exists whithin your Form Images library, for a user's reference when completing the field. example: Insert a Risk Assessment Matrix that is necessary for a user's visual referece to complete a JSA.

NINTH COLUMN - choices

A drop-down options for the select field.

  • Separate choices with a semicolon (;)
  • note: When inputting a select, rig-select, people-select or lookup-select field type; you must input the corresponding is_multi choice -> TRUE or FALSE

TENTH COLUMN - lookup-list

Specify the lookup listname that exists within your Form Lookup Lists library.

  • To write something here, the type must be lookup-select.

EXTRA COLUMN - no name
